Extending the Blender Plugin

Civic Builder thrives on community contributions! Whether you're a developer, designer, or simply enthusiastic about enhancing Blender workflows, this page outlines how to contribute extra features to the plugin.

Getting Started

To contribute to the Civic Builder plugin, you should be familiar with Blender scripting, particularly using Python. Before you begin, ensure you have:

  • A working installation of Blender (version 3.x or newer).
  • Basic knowledge of Blender's Python API.
  • Git installed for managing version control.

How to Contribute Extra Features

Follow these steps to add new features to the Civic Builder plugin:

1. Fork the Repository

Start by forking the official Civic Builder GitHub repository. Clone the repository locally using the following command:

git clone https://github.com/your-username/civic-builder.git

2. Set Up Your Development Environment

Install the plugin in Blender for testing. Follow these steps:

  1. Open Blender and go to Edit > Preferences > Add-ons.
  2. Click Install and select the .zip file of the plugin.
  3. Enable the plugin to ensure it loads correctly.

3. Add Your Feature

Add your custom functionality to the plugin. Here's a typical structure of the plugin's codebase:

  • civic_builder.py: The main entry point for the plugin.
  • operators/: Contains operator definitions for different actions.
  • ui/: Defines the user interface panels and menus.
  • utils/: Helper functions for shared functionality.

Create a new operator in the operators/ directory for your feature. Here's an example template for an operator: 

    import bpy
    
    class OBJECT_OT_new_feature(bpy.types.Operator):
        """Tooltip for your new feature"""
        bl_idname = "object.new_feature"
        bl_label = "New Feature"
        bl_options = {'REGISTER', 'UNDO'}
    
        def execute(self, context):
            # Your feature logic goes here
            self.report({'INFO'}, "New feature executed successfully!")
            return {'FINISHED'}
    
    def register():
        bpy.utils.register_class(OBJECT_OT_new_feature)
    
    def unregister():
        bpy.utils.unregister_class(OBJECT_OT_new_feature)

4. Test Your Feature

Thoroughly test your new feature in Blender to ensure it works as expected. Debug any issues you encounter.

5. Submit a Pull Request

Once your feature is ready, commit your changes and push them to your forked repository. Open a pull request to the main repository with a detailed explanation of your changes.

git add . git commit -m "Added new feature: [Feature Name]" git push origin main

Ideas for Extra Features

If you're looking for inspiration, here are some ideas for features to contribute:

  • New building types or templates.
  • Additional procedural controls for geometry nodes.
  • Performance optimization features for large-scale city generation.
  • Export functionality to popular game engines like Unity or Unreal Engine.
Tip: Join the Civic Builder community to discuss ideas, get feedback, and collaborate with other contributors.
