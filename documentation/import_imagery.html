Importing Imagery

Before extracting features, the user must set input and output paths for Civic Builder to use. The following paths are required:

  • • Input Directory: The directory containing the imagery files to be processed.
  • • 2D Output Directory: The directory where the 2D output image will be saved.
  • • 3D Output Directory: The directory where the 3D output files will be saved.

Step 1: Launch Civic Builder Standalone

After installing the application, start Civic Builder by launching Civic Builder.exe from the installation directory on your computer.

Step 2: Navigate to Input/Output Window

Within the GUI, navigate to the "Input/Output Paths" window to manage your input and output directories. Note that this is also the default window. Your preferences will be saved after running the application to avoid resetting each time.

Step 3: Select Input Directory

Select your input directory containing .JPG or .PNG image files. Consider the following when selecting import imagery:

  • • Images are not distorted.
  • • Image sizes are in multiples of 256 pixels. So 512,1024,2048,4096 are preferred but no mandatory.
  • • Images are not of incompatible types like .WEBP or .TIFF.
  • • Images do not contain significant discoloration or heavy filters.
  • • Images are not zoomed in or out too much.
  • • Ensure the imagery resolution matches the recommended resolution provided in the GUI settings.

Step 4: Set Output Directories

Specify the output directories for:

  • • 2D Annotated Output: Annotation of detected buildings on a 2D image with confidence metrics per building.
  • • 3D Object Output: Directory where the footprint .obj and imagery .obj polygon files will be saved.

Step 5: Ready to Process

After setting up your input and output directories, you're ready to proceed with selecting preferences and initiating the detection process. Click the green button "Proceed to Run Extraction" to quickly progress to the extraction screen.

Common Issues

Be aware of the following common issues at this stage of the workflow:

  • • Corrupted files in the input directory are not accepted. Make sure that input images are valid by opening in your prefered image manipulation software such as Adobe Photoshop.
  • • Files in formats other than JPG or PNG (e.g., TIFF or WEBP) are not supported.
  • • Media of insufficient resolution may not produce accurate results. Make sure to select the correct model for your resolution and imagery type to achieve the best results.
