Post Processing

The post-processing step in Civic Builder enhances the quality and usability of generated 3D footprints. The tool automatically applies several optimization techniques to refine the raw data extracted during the detection process. Users can leverage these built-in features to clean, simplify, and structure the output for downstream applications like GIS tools, game engines, or simulation software.

For example, some users may not need the extra details and may opt for the bounding boxes algorithm to simplify the output. Others may require more detailed contours and choose to smooth the contours for a more refined output.

Common Issues and Solutions

  • Contours Too Simplified: Select the simplify contours algorithm, which will do a simple pass and retain most geometry.
  • Contours Too Complex: Ensure input imagery aligns with the resolution specified in the detection step.
  • Loss of Details: In some cases, post processing may remove important details with options such as "Bounding Boxes" or "Smooth Contours". In this case it is best to switch to the default "Simplify Contours" algorithm.

How to Enable Post Processing

  1. In the **Preferences** window, select the desired post-processing algorithms you wish to apply.
  2. Ensure the options are tailored to your output needs (e.g., select "Fill Holes" for fragmented datasets).
  3. Run the detection process. Post-processing will automatically apply to the output before saving.

Available Post-Processing Options

Civic Builder offers the following inbuilt post processing algorithms:

Simplify Contours

Default mode. This function reduces the number of points in a contour, preserving the overall shape while minimizing unnecessary details. Simplification ensures smaller file sizes and faster processing in 3D environments.

Smooth Contours

Smoothing approximates contours with fewer points, reducing jagged edges in 3D models. Users can control the level of smoothing via the epsilon factor for precise customization.

Fill Holes

Automatically fills gaps or holes in binary masks, producing cleaner and more complete building footprints. This step is essential when dealing with fragmented data or noisy imagery.

Generate Bounding Boxes

Automatically creates bounding boxes around detected objects, providing spatial boundaries for buildings or other features. Bounding boxes are ideal for GIS mapping and object labeling.

Convex Hulls

Computes the convex hull for each contour, ensuring no concave regions. Convex hulls are particularly useful for geometric simplifications in architecture and simulation.
